# Lab 10: Configuring NAT and Using Wireshark

**Module:** M10 | **Course:** IT158 | **Date:** October 2025

## Tools Used
- Windows Server 2019
- Wireshark
- NDG NetLab

## What I Did
Configured Network Address Translation on a Windows Server 2019
environment and used Wireshark to capture and analyze the resulting
network traffic. Verified NAT was functioning correctly by examining
packet captures and confirming address translation was occurring
as expected.

## Key Concepts
- NAT configuration on Windows Server 2019
- Wireshark packet capture and analysis
- Verifying NAT through traffic inspection
- Source and destination address translation
- Network traffic analysis methodology

## Why This Matters
Combining NAT configuration with Wireshark analysis demonstrates
both server administration and network analysis skills together.
Being able to verify what a server is actually doing at the packet
level — not just assuming it works — is a skill that separates
strong candidates from average ones in server and network roles.
